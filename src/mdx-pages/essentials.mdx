<p className="pb-3 pt-3">Voici quelques explications concernant les grandes étapes du développement d’une Single Page Application (SPA) :</p>

<h4 className="pb-1">1. Création d’une version préliminaire du frontend</h4>
<p className="pl-5">
  Cette version du frontend reprend tout la mise en page souhaitée (header,
  footer, main, barre de navigation…) et permet d’afficher les différentes pages
  qui, généralement, sont juste une version statique de celles-ci. A cette
  étape, la mise à jour dynamique du frontend via des opérations sur des Web
  APIs n’est pas encore envisagé. N’hésitez pas néanmoins à cette étape-ci à
  rendre votre frontend dynamique pour tout ce qui peut être fait côté client
  (affichages des bons composants lors de clics, animations…).
</p>
<h4 className="pt-3 pb-1">2. Création de la RESTful API</h4>
<p className="pl-5">
  Cette Web API va mettre à disposition des opérations sur des ressources (les
  données) nécessaires pour rendre votre site web dynamique.
</p>
<h4 className="pt-3 pb-1">
  3. Création de votre Single Page Application (SPA)
</h4>
<p className="pl-5">
  Cette étape consiste à mettre à jour votre frontend afin que celui-ci
  s’intègre avec votre RESTful API. Votre site devient dynamique sur base
  d’opérations sur des ressources offertes par votre RESTful API.
</p>

<p className="pt-3 pb-3">
  Nous espérons que cette série vous facilitera le développement de votre propre
  site.
</p>
